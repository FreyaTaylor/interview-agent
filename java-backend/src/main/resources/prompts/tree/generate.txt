## 要求
1. 根据需求生成合理的层级结构（2-4层深度）
2. 一级分类 3-10 个，覆盖需求描述的核心领域
3. 叶子节点是具体的知识点/面试考点，名称简短（≤15字）
4. 每个叶子节点标注面试权重 interview_weight（1-5）
5. 数量合理，不凑数，核心考点优先
6. 高频考点靠前排列

```json
          ]

```
